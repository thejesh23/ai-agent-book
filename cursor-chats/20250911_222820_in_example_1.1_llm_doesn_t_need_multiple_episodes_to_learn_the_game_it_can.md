Interestingly, however, a Q-learning-based agent training for 10,000 episodes can take less time than an LLM agent playing a single episode. In contrast, the LLM agent requires the LLM to think and reason at every step, making a single episode very time-consuming. Therefore, if simulation is efficient and the task is simple, traditional RL may be faster. But in the real world, many tasks have high simulation costs and low efficiency—such as web search, operating a browser, making phone calls, or moving a robot in the physical world—all of which are far more expensive than this simple game. Thus, the sample-efficient LLM agent is better suited for these complex scenarios.

In Example 1.1, the LLM does not require multiple episodes to learn this game. Even without prior experience, it can win in as few as 17 steps in a single run, reasoning at each step, making decisions based on common sense, and recording key discoveries in its context to leverage in-context learning. A traditional RL agent can find the optimal 11-step solution, but it requires nearly 10,000 episodes of exploration. This highlights a critical difference in learning efficiency (sample efficiency).Why? Because at the end of the corridor there is a door, a key opens the door, a sword kills the guard, and humans have concepts. But in Q-learning, these are all meaningless A, B, C.

Interestingly, however, a Q-learning-based agent takes longer to train for 10,000 episodes than an LLM agent takes to play a single game. This is because this game is very simple, the simulation efficiency is high, Q-learning can easily learn a table of just over 100 states to represent the entire game, and convergence is fast. But an LLM agent needs the LLM to think and reason each time, so playing one game takes a long time. Therefore, if simulation efficiency is high and the task is simple, traditional RL may be faster. However, many tasks in the real world have high simulation costs and low efficiency, such as web search, operating a browser, making phone calls, or moving a robot in the physical world. These are all much more costly than this small game, so sample-efficient LLM agents are better suited for these complex scenarios.
