## 📊 Relationship Between DAPO and GRPO

**GRPO** (Group Relative Policy Optimization) is the foundational algorithm proposed by DeepSeek.

**DAPO** (Decoupled Clip and Dynamic Sampling Policy Optimization) builds upon GRPO by adding **three key improvements**.

## 🔍 Three Major Improvements of DAPO over GRPO

### 1. **Decoupled Clipping Parameters** (Separated Clip Epsilons / Clip-Higher)

**GRPO (Symmetric Clipping)**:
```python
# Standard PPO/GRPO: upper and lower bounds are the same
pg_losses = -advantages * torch.clamp(ratio, 1 - ε, 1 + ε)
# clip_ratio = 0.2, i.e., [0.8, 1.2]
```

**DAPO (Asymmetric Clipping)**:
```python
# DAPO: upper bound is more lenient
pg_losses1 = -advantages * ratio
pg_losses2 = -advantages * torch.clamp(ratio, 1 - ε_low, 1 + ε_high)
pg_losses = torch.maximum(pg_losses1, pg_losses2)
# clip_ratio_low = 0.2, clip_ratio_high = 0.28
# i.e., [0.8, 1.28]
```

**Configuration Comparison**:
```yaml
# GRPO
clip_ratio: 0.2  # Both upper and lower bounds are 0.2

# DAPO
clip_ratio_low: 0.2   # Lower bound
clip_ratio_high: 0.28  # Upper bound is more lenient
```

**Impact**: Allows the model to take larger update steps for good behaviors (positive advantages), accelerating learning.

### 2. **Dynamic Sampling with Group Filtering**

**GRPO**:
- Fixed sampling: generates n responses per question
- All groups are used for training

**DAPO**:
```yaml
data:
  gen_batch_size: 1536      # Generation batch size
  train_batch_size: 512     # Training batch size
algorithm:
  filter_groups:
    enable: True
    metric: acc              # Filter by accuracy
    max_num_gen_batches: 10  # Generate at most 10 batches
```

**How it works**:
```python
# DAPO filters out "uninformative" groups
# For example: all 16 responses to a question are correct or all wrong → filtered out
# Only groups with discriminative power (some correct, some wrong) are kept for training
```

**Advantages**:
- Improves sample quality, avoiding wasted computation on uninformative samples
- Dynamically adjusts sampling until enough high-quality samples are collected
- In your experiment, DAPO may have generated more samples but only used the most valuable 512

### 3. **Token-level Loss Aggregation**

**GRPO (Sequence-level)**:
```python
# Original GRPO: seq-mean-token-mean
seq_losses = torch.sum(loss_mat * loss_mask, dim=-1) / torch.sum(loss_mask, dim=-1)
loss = torch.mean(seq_losses)  # Average the average loss of each sequence
```

**DAPO (Token-level)**:
```python
# DAPO recommendation: token-mean
loss = verl_F.masked_mean(loss_mat, loss_mask)  # Average directly over all tokens
```

**Configuration**:
```yaml
actor_rollout_ref:
  actor:
    loss_agg_mode: "token-mean"  # DAPO default
    # vs "seq-mean-token-mean"  # Original GRPO
```

**Advantages**:
- More stable for long sequences (common in mathematical reasoning with long CoTs)
- Prevents short sequences from dominating the gradient
- Provides more fine-grained learning signals

### 4. **Overlong Response Penalty**

**DAPO additionally includes**:

**Purpose**: Encourage the model to generate concise and effective reasoning, avoiding lengthy and unproductive exploration.
