# Experiment 5-5: Automatic Generation of Paper Explanation Videos ★★

Building on "Paper → PPT", the Agent generates **colloquial narration** (guiding narrative,
rather than a point-by-point recitation) for each slide, calls **TTS** to synthesize speech,
and then uses **ffmpeg** to **synchronize and composite** the PPT screenshots with the audio
**page by page** into a narrated explanation video.

## Workflow

```
Paper Key Points (built-in example)
   │  PIL Rendering
   ▼
Per-page PNG Slides ──► gpt-5.6-luna Generates Colloquial Narration ──► OpenAI tts-1 Synthesizes mp3
   │                                                        │
   └──────────────── ffmpeg: Per-page PNG + Page Audio ─────┘
                              │  (Page Duration = Audio Duration)
                              ▼
                     ffmpeg concat Stitch
                              ▼
                     output/lecture.mp4
```

- This project is **self-contained**, independent of Experiment 5-4: it includes a built-in set of key points from the paper "Attention Is All You Need",
  and uses PIL to directly render 5 slide PNGs (can also be replaced with Slidev screenshots from 5-4).
- The narration is generated by `gpt-5.6-luna`; the speech is synthesized by OpenAI `tts-1` (`voice=alloy`).
- The video is composited by `ffmpeg`: each page is made into an mp4 segment, with the segment duration equal to the audio duration of that page, and finally concatenated,
  thus ensuring **precise alignment between each page's display time and its speech duration**.

## Run Commands

```bash
pip install -r requirements.txt      # Install Python dependencies
cp env.example .env                  # Fill in OPENAI_API_KEY (if not configured, OPENROUTER_API_KEY will be used as a fallback for narration generation; TTS will degrade to offline placeholder)
python demo.py                       # Generate the complete explanation video for all 5 pages
```

Common parameters (see `python demo.py --help` for all):

```bash
python demo.py --check     # Environment self-check: check ffmpeg/ffprobe/fonts/configuration, no API calls
python demo.py --quick     # Quick smoke test: run only page 1 (equivalent to --limit 1), saves time and money
python demo.py --limit 2   # Process only the first 2 pages
python demo.py --offline   # No API needed: placeholder silent audio track, verify the entire ffmpeg compositing pipeline
```

Full parameters:

| Parameter | Description |
| --- | --- |
| `--slides FILE` | Slide content JSON (`[{title, subtitle, bullets}, ...]`), replaces the built-in example |
| `--script FILE` | Ready-made narration JSON (list of strings, one paragraph per page); if provided, **skips LLM generation** |
| `-o, --output FILE` | Final video output path (default `output/lecture.mp4`) |
| `--tts-provider {openai,offline}` | TTS provider; `offline` uses ffmpeg to generate a placeholder silent audio track (no API needed) |
| `--offline` | Fully offline: equivalent to `--tts-provider offline`, and uses key points as placeholder narration (zero API calls) |
| `--text-model / --tts-model / --tts-voice` | Override model/voice (defaults to environment variables of the same name) |
| `--limit N / --quick / --check` | Run only the first N pages / run only page 1 / self-check only |

> **Offline Verification**: `--offline` requires no API Key or network. It uses `ffmpeg anullsrc` to estimate duration based on narration word count and synthesize a silent placeholder audio track, running through the entire pipeline of "Rendering → Duration Estimation → Per-page Compositing → Concat Stitching". It is specifically designed to verify whether ffmpeg's **per-page duration alignment** is correct (the audio track is a silent placeholder, not real dubbing).

Artifacts:
- `output/slides/slide_*.png`   Each slide
- `output/audio/audio_*.mp3`    Each page's narration audio
- `output/segments/seg_*.mp4`   Each page's segmented video
- `output/narration.json`       List of narration and audio durations for each page
- `output/lecture.mp4`          Final explanation video

View video metadata:

```bash
ffprobe -v error -show_format -show_streams output/lecture.mp4
```

## Expected Output Example

Using the built-in 5-page "Attention Is All You Need" as an example, the actual artifacts from a complete run:

- `output/lecture.mp4`: Approximately **2.8 MB**, duration **166.97s** (≈2 minutes 47 seconds),
  resolution **1280×720**, video **H.264** + audio **AAC**.
- `output/audio/audio_01.mp3 … audio_05.mp3`: One narration per page,
  individual page durations approximately **28.6s / 33.3s / 37.2s / 37.9s / 29.9s** (total ≈166.9s, consistent with the video duration).
- `output/narration.json`: The colloquial narration and audio duration list for each page, for example, page 1:

  > Today, we are going to explore an important study that has transformed the field of natural language processing—"Attention Is All You Need"……
  > It relies entirely on the attention mechanism, discarding traditional recurrent and convolutional structures.

The run log will print "Slide → Narration → Audio Duration" page by page, and at the end, summarize the total audio duration of all pages and the final video duration
(the two should be basically consistent, indicating that the display time of each page is precisely aligned with the speech).

## Dependencies

- **ffmpeg / ffprobe**: Command-line tools (verified with version 8.x for this project). On macOS, you can `brew install ffmpeg`.
- **Python packages**: `openai`, `Pillow`, `python-dotenv` (see `requirements.txt`).
- **Chinese fonts**: Rendering slides requires a Chinese font on the system. The script automatically falls back to common macOS fonts
  (PingFang / STHeiti / Hiragino / Arial Unicode).
- **Environment variables**: Requires `OPENAI_API_KEY` (for the official OpenAI API); if not configured, `OPENROUTER_API_KEY` can be used as a fallback for narration generation (in this case, TTS will degrade to an offline silent placeholder because it is not on OpenRouter). Optional items can be found in `env.example`.

## How to Adapt / Extend

- **Change Model / Provider**: Can be done via environment variables or command line, no code changes needed:
  - `TEXT_MODEL` / `--text-model`: Narration generation model (default `gpt-5.6-luna`, can be changed to others).
  - `TTS_MODEL` / `TTS_VOICE` (or `--tts-model` / `--tts-voice`): Speech model and voice
    (default `tts-1` / `alloy`, other voices include `nova` / `shimmer` / `echo`, etc.).
  - `--tts-provider offline`: Switch to offline placeholder audio track (generates no API calls), used for local verification.
  - `OPENAI_BASE_URL`: Point to any **OpenAI-compatible** custom endpoint (self-built gateway, proxy, or
    third-party provider); use the corresponding `OPENAI_API_KEY` to switch backends.
- **Change Input (Change Paper / PDF)**: Use `--slides my.json` to pass in external slide content, or directly edit
  the `SLIDES` list in `demo.py` (title / subtitle / key points); if you already have a real PDF, you can first use the
  "Paper → PPT" workflow from 5-4 to produce key points or Slidev screenshots, then feed them to this script; the rest of the workflow remains unchanged.
- **Provide Your Own Narration**: Use `--script narr.json` (a list of strings, one paragraph per page) to skip LLM generation
  and go directly to "TTS → Compositing", which is convenient for re-running after manually polishing the script.
- **Longer Videos**: Increase the number of `SLIDES` pages or lengthen the narration for each page (a single run can be 5~15 minutes).
- **Quick Parameter Tuning**: First use `--quick` / `--limit N` to render only a few pages, confirm satisfaction with the voice/style, then run the full set.

## Limitations

- Both narration generation and TTS will incur real OpenAI API calls (`TEXT_MODEL` and `TTS_MODEL`), which will **incur costs**;
  the full 5 pages will generate approximately 2~3 minutes of video. It is recommended to first run `--check` for self-checking, then `--quick` for a smoke test.
- The slides are purely static PIL renderings (no animations/transitions), and Chinese font rendering depends on system fonts. Non-macOS users may need to adjust
  `FONT_CANDIDATES` themselves.
- The duration of each page is strictly equal to the audio duration of that page; there are no silent pauses or background music. For more refined layout and transitions,
  it is recommended to use Slidev screenshots from 5-4 as input.
